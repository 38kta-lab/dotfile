# `ja.html` テンプレート

`ja.html` は、論文本文の節構造を土台にしつつ、日本語で再読しやすい精読メモにする。

## 基本構造

- `Abstract`
  - 論文の核を 2-3 段落で言い直す
- `導入の要点`
  - 背景
  - 何が未解決か
  - この論文が何をしようとしているか
- `Materials and methods 日本語整理`
  - 小節ごとに 1-2 段落で整理
  - 何を入力にし、何を測り、何を比較したかを残す
- `Results and discussion 日本語整理`
  - 論文の節順に従う
  - 図の役割と、そこから直接言えることを分ける
- `Discussion の要点`
  - 論文全体のメッセージを再統合する
- `Conclusions の要点`
  - 最終主張を短く言い直す
- `批判的コメント`
  - モデル依存性
  - 実験の弱さ
  - 一般化の限界
  - 代替解釈
- `次に読むポイント`
  - 次回どこを深掘りするかを箇条書き

## 書き方のルール

- まず「何をしたか」より「この節の役割は何か」を先に書く
- `Fig. X` は「何を示す図か」を 1 文で固定する
- データから直接言えることと、著者のモデルや解釈を混ぜない
- 必要なら `Red2 だけ外れる` のような例外条件を明示する
- 断定を避けるべき箇所はそう書く

## 最低限残すべき批判点

- 何がモデル由来の推定か
- どこに実験的未検証が残るか
- どの条件で予測が外れているか
- 代替解や別説明の余地があるか


## 完成条件

- `ja.html` は最終的に self-contained HTML にする
- ローカル図がある場合、`src` だけでなく `srcset` も埋め込む
- 単体で別PCや Desktop に持っていっても図が見える状態を完成形とする

## `clean-original.html` 前提

- `ja.html` は raw HTML ではなく `clean-original.html` を土台にして書く
- `clean-original.html` は publisher ごとに DOM が違ってもよい前提で扱う
- `article` / `main` / publisher-specific body のどれを起点にしたかを意識し、本文外の UI を混ぜない

## 見た目の方針

- `ja.html` は `daily-search-trend` の Newsprint 系 CSS を基盤にする
- 背景は紙面寄り、本文は serif、見出しは強すぎない罫線中心でまとめる
- `批判的コメント` や `次に読むポイント` だけは補助ボックス化してよいが、全体は新聞紙面の延長で揃える
